Git入门

2022-11-24 3 分钟阅读 后端JavaGit

Git入门（以前的知识突然袭击我）

  • 用户名和邮箱

    1git config --global user.name "your name"
2
3git config --global user.email "email address"
4
5# 上面这种是全局设置，如果只是对项目级别设置，去掉--global 即可
6

  • 编辑器

    git默认的编辑器是GNU nano这样的编辑器，我可以通过如下的命令把它设置为vim编辑器

    git config –global core.editor vim

  • 命令

    常用命令

 1    # init：初始化项目，进入到项目的文件夹内，使用如下命令进行初始化
 2    # 执行完成后，会在当前目录下生成.git的隐藏文件夹
 3    # 目录也不再是普通的文档目录了，今后我们将其称为 工作树
 4    git init 
 5
 6    # 执行下面的命令，Git 会自动调用系统默认的文本编辑器，要求你输入版本更新说明并保存。
 7    git commit
 8    
 9    # 或者这样，直接把命令和说明一起提交
10    git commit -m "版本更新的信息"
11
12    # 得到一个项目的拷贝
13    #  在默认情况下，Git会把"Git URL"里最后一级目录名的'.git'的后辍去掉,做为新克隆(clone)项目的目录名: 
14    # (例如. git clone git@github.com:spring-projects/spring-framework.git 会建立一个目录叫'spring-framework' )
15    git clone
16    
17    # 如果访问一个Git URL需要用法名和密码，可以在Git URL前加上 用户名 ，并在它们之间加上 @ 符合以表示分割，
18    # 然后执行git clone命令，git会提示你输入密码。
19    # git clone old.ray @ git@github.com:spring-projects/spring-framework.git
20
21    # 可以通过-b <name>来指定要克隆的分支名
22    git clone -b branchname git@github.com:spring-projects/spring-framework.git
23
24    # git add 命令所生成的快照被存放到一个临时的 存储区域 ，Git 称该区域为 索引。
25    # 添加单个文件
26    git add 
27    
28    # 如果不记得添加了哪些文件，可以用如下命信，这里的 .表示当前目录。
29    # 用.方式除了能够判断出当前目录（包括其子目录） 所有被修改或者已删除的文档，还能判断 用户所添加的新文档，并将其信息追加到索引中。
30    git add .
31
32    # 通过 git push把本地仓库的更新推到服务器仓库
33    git push
34   

其他命令

 1    # 查看版本历史
 2    # 在工作树中，使用git log 命令可以查看当前项目的日志，也就是你在使用git commit 向仓库提交新版本时所属如的版本更新信息。
 3    git log
 4
 5    # 想看一下每一次版本的大致变动情况，可使用以下命令
 6    git log --stat --summary
 7
 8    # git log 每次提交都会有一个唯一的版本号，使用下面的命令可以查看提交的具体情况
 9    git show commitId(版本号)
10
11    #撤销
12    #撤销未提交的修改( git reset)
13    # 这条件命令会把你所以工作目录中所有未提交的内容清空(当然这不包括未置于版控制下的文件 
14    git reset --hard  HEAD
15
16    # 如果你只是要恢复一个文件,如"hello.rb", 你就要使用git checkout
17    git checkout --  hello.rb
18
19    # 如果你想要恢复当前目录所有修改的文件,你可以使用
20    git checkout .
21

.gitignore说明

在通过git add 命令生成文档内容快照时，工作树中有一些文档是你不希望接受Git 管理的，譬如程序编译时生成的中间文件，对于这样的文件如何避免为之生成快照？

为解决此类问题，Git 提供了文档忽略机制，可以将工作树中你不希望接受Git 管理的文档信息写到同一目录下的.gitignore 文件中。

比如对于project目录下的out目录，可以采用如下操作可将其排除到git管理之外.

目录:

public/

test/

文件：

*.log

a.txt
