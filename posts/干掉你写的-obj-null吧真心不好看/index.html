干掉你写的 Obj != Null吧，真心不好看

2021-06-04 2 分钟阅读 后端Java

前言：在平时开发过程中，字符串的非空判断时有发生，我们最常用到的a != null && !"".equals(a) 虽然没啥问题，但是真心不好看，这里有一些其他的非空校验方法希望可以帮到你。

一，StringUtils类

 1//该类提供了一系列关于字符串类型的操作方法，如非空校验，包含判断等
 2//引入依赖使用：
 3<dependency>
 4    <groupId>commons-lang</groupId>
 5    <artifactId>commons-lang</artifactId>
 6    <version>2.6</version>
 7</dependency>
 8
 9//原来的写法：
10if(a == null || "".equals(a)){
11	System.out.println("a为空的操作");
12}
13//可以改写成：
14if(StringUtils.isBlank(a)){
15	System.out.println("a为空的操作");
16}
17
18//常用的方法：
19StringUtils.isNotBlank：非空校验，就算全部是空格也会返回false,即也是空
20StringUtils.isNotEmpty：非空校验，如果全部是空格会返回true,即认为有值
21
22StringUtils.isNotBlank(null)      = false
23StringUtils.isNotBlank("")        = false
24StringUtils.isNotBlank(" ")       = false   //纯空格仍旧认为是空数据
25StringUtils.isNotBlank("bob")     = true
26StringUtils.isNotBlank("  bob  ") = true
27
28StringUtils.isNotEmpty(null)      = false
29StringUtils.isNotEmpty("")        = false
30StringUtils.isNotEmpty(" ")       = true    //纯空格会认为是非空
31StringUtils.isNotEmpty("bob")     = true
32StringUtils.isNotEmpty("  bob  ") = true
33
34//当然还有取反的方法
35StringUtils.isBlank
36StringUtils.isEmpty
37
38//这个类里面还有很多其他的方法，有兴趣可以看下。

二，MapUtils类

 1//该类可以对Map的value直接指定数据类型返回，也可以同时设置默认值，非常好用。
 2//引入依赖使用：
 3<dependency>
 4    <groupId>org.apache.commons</groupId>
 5    <artifactId>commons-collections4</artifactId>
 6    <version>4.1</version>
 7</dependency>
 8
 9Map<String,Object> map = new HashMap<>();
10map.put("name","张三");
11map.put("age",18);
12map.put("amount",0.01);
13
14//原有的写法太过于复杂很容易看错写错：
15String name = (map.get("name")==null)?"":(String)map.get("name");
16Integer age = (map.get("age")==null)?0:(Integer) map.get("name");
17Double amount = (map.get("amount")==null)?0d:(Double) map.get("amount");
18
19//使用工具类：
20//获取key=name字符串类型数据，为空返回名字。
21MapUtils.getString(map,"name","名字");   
22
23//获取key=age整形数值，为空返回0
24MapUtils.getInteger(map,"age",0);
25
26//获取k=amount双精度数值，为空返回0d
27MapUtils.getDouble(map,"amount",0d);
28

总结，上面这两个工具类真心好用，虽然有损耗性能的嫌疑，但我想说性能固然重要，但能优雅地写代码取乐自己不也很重要嘛。我是阿雷，一个越来越胖的程序员。
