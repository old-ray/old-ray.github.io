Anaconda 安装

参考链接：

https://blog.csdn.net/wyf2017/article/details/118676765

https://zhuanlan.zhihu.com/p/94744929

去官网下载想要的版本：

https://repo.anaconda.com/archive/

一般下载sh文件直接运行就可以了，Linux环境使用wget命令即可，例：

wget https://repo.anaconda.com/archive/Anaconda3-2022.10-Linux-x86_64.sh

安装步骤：

赋权限：chmod +x Anaconda3-2022.10-Linux-x86_64.sh

运行脚本：./Anaconda3-5.3.0-Linux-x86_64.sh

虚拟环境

创建（workspace_doc 是我自己取的名字，也可以不指定python版本）

conda create -n workspace_doc python=3.7

激活环境

conda activate workspace_doc

上面命令如果报错，使用

source activate workspace_doc

列出所有环境

conda env list

关闭虚拟环境

conda deactivate

删除虚拟环境

conda remove -n workspace_doc --all

jupyter lab安全验证

参考链接：

https://blog.csdn.net/qq_34419607/article/details/109177125

https://blog.csdn.net/qq_27370437/article/details/117845115

生成配置文件（注意notebook的配置文件过期了，用server）

jupyter server --generate-config

该命令会在 ~/.jupyter 目录下生成配置文件 jupyter_server_config.py, 如果该配置文件已经存在，则会提示是否替换该文件。

设置密码

jupyter server password

该命令会在 ~/.jupyter 目录下生成配置文件 jupyter_server_config.json，复制里面的命码

允许后台运行

nohup jupyter lab --allow-root > jupyterlab.log 2>&1 &

最后将域名映射到服务器的端口上，输入密码即可访问，最后是申请SSL安全证书。

最后非常重要，要配置Nginx!!!

因为jupyter有wss协议请求，如果nginx不配置的话，会提示连接重置 **Replacing stale connection:** 网上搜的答案大部份说是tornado 5.1.1降级，但是我降级提示tornado5.1.1这个版本就不支持了，因为我的jupyter本身版本就很高，这个问题我处理了很久，结果是路子找错了，方向不对所以一直困在这里，于是我就按上面的另一个错误提示400 GET /terminals/找，才晓得是nginx的问题，于是我打开浏览器的调试页面，发现有些链接是wss协议的，而我的nginx没有配置，所以请求就没响应，于是我把下面的两个配置加好就ok了，很明显第一个是python计算用的，第2个是打开控制台用到。至此这个问题才终于解决了，很坑！！！

 1location ~ /api/kernels/ {
 2    proxy_pass http://127.0.0.1:8888;
 3    proxy_http_version 1.1;
 4    proxy_set_header Upgrade $http_upgrade;
 5    proxy_set_header Connection "Upgrade";
 6    proxy_set_header Host $host;
 7}
 8
 9        location ~ /terminals/ {
10    proxy_pass http://127.0.0.1:8888;
11    proxy_http_version 1.1;
12    proxy_set_header Upgrade $http_upgrade;
13    proxy_set_header Connection "Upgrade";
14    proxy_set_header Host $host;
15}

jupyter 安装git 插件

注意git的版本2.0+，当时就是因为git版本只有1.8导致一直安装不上，更新git的步骤参考

https://www.jianshu.com/p/89a2917ff5c1

猜想：最开始编译失败的原因是因为下面的命令没有执行

yum install curl-devel expat-devel gettext-devel openssl-devel zlib-devel gcc perl-ExtUtils-MakeMaker

git插件我安装到虚拟环境里了，要使用的话先进入虚拟环境，页面上才会出现git工具，吃过亏，直接在虚拟环境打开了jupyter发现安装的git又不见了，搞得重新装了一次。一定要进入虚拟环境再启动jupyter。
